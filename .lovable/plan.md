
الهدف: معالجة التهنيج الشديد أثناء المعاينة والتسجيل، ومنع توقف الفيديو مع استمرار الصوت، وتثبيت تسجيل خلفيات الفيديو (خصوصًا Pexels) مع إخراج سلس على الأجهزة الضعيفة والقوية.

1) نتائج الفحص السريع (المشكلة الحالية)
- مسار التسجيل الحالي ما زال يعمل على نفس الـ Main Thread؛ لذلك عند ارتفاع تكلفة الرسم يحصل تجميد للواجهة والتسجيل معًا.
- جودة التصدير الحالية تؤثر غالبًا على الـ bitrate فقط، بينما أبعاد الرسم أثناء التسجيل تبقى كبيرة في أغلب الأوضاع؛ هذا يفسر “فيديو ثقيل” حتى عند اختيار جودة أقل.
- حلقة الرسم أثناء التسجيل تعتمد على مرجع تم التقاطه عند بدء التسجيل؛ هذا قد يؤدي إلى Frames لا تُحدّث محتوى الآيات/الحركة بشكل صحيح أثناء تقدم الصوت.
- رسم كل عناصر النص والزخارف يتم كل Frame (قياسات نص + مؤثرات + أشكال)، وهذا مكلف جدًا على الموبايل.
- تم رصد فشل تحميل خلفيات فيديو غير Pexels في الاختبار (403 + fallback)، وهذا يزيد عدم الاستقرار عند الخلفيات البديلة.

2) خطة الإصلاح (تنفيذ متدرج وآمن)
المرحلة A — “إصلاحات حرجة” فورية لتخفيف الحمل
- ربط أبعاد التسجيل فعليًا بمستوى الجودة (low/medium/high/ultra) بدل الاعتماد على bitrate فقط.
- جعل وضع auto يبدأ بمحرك “compatibility” للأجهزة الضعيفة، ثم يصعد للأعلى فقط عند ثبات الأداء.
- إصلاح مرجع الرسم أثناء التسجيل بحيث يسحب أحدث دالة رسم حيّة (بدون stale closure) لضمان استمرار تحديث الفيديو/الآيات.
- وضع سقف آمن تلقائي للحمل أثناء التسجيل:
  - تعطيل/تخفيف الجزيئات والزخارف الثقيلة تلقائيًا عند رصد هبوط FPS.
  - تخفيف الظلال والـ glow والعمليات المكلفة في التسجيل قبل أن يصل المتصفح للتجميد.

المرحلة B — إعادة هندسة الرسم للأداء (Core Performance Refactor)
- تقسيم الرسم إلى طبقات:
  1) طبقة خلفية متحركة
  2) طبقة عناصر ثابتة نسبيًا (إطار/اسم السورة/اسم القارئ)
  3) طبقة نص الآية المتغير
- عمل caching لنتائج layout للنص (line wrapping + measureText) وعدم إعادة الحساب إلا عند تغير الآية/الخط/المقاس.
- تحديث React state الخاص بالوقت/التقدم بتردد أخف أثناء التسجيل (throttled UI updates) لتقليل إعادة تصيير الصفحة كاملة.
- إبقاء الرسم الداخلي للتسجيل منفصلًا عن Canvas المعاينة مع تحكم frame pacing ثابت.

المرحلة C — تثبيت مسار خلفيات الفيديو (Pexels أولًا)
- تحسين اختيار رابط فيديو Pexels ليكون “مناسبًا للجهاز” (ليس أعلى دقة دائمًا)، مع تفضيل مقاس متوازن للتسجيل العمودي.
- تحسين مسار proxy الخلفي ليكون أخف (streaming/range-aware) بدل تحميل الملف كاملًا في الذاكرة عندما يلزم.
- توحيد منطق fallback بحيث لا يحدث flicker (الانتقال المباشر إلى fallback واضح وثابت).
- إبقاء مؤشر حالة الخلفية واضحًا (Direct / Proxy / Fallback) مع سبب مبسط عند الفشل.

المرحلة D — وضع عزل تسجيل متقدم (Background Export Mode)
- إضافة وضع “تسجيل بالخلفية” داخل التطبيق:
  - إيقاف أغلب تحديثات UI غير الضرورية أثناء التسجيل.
  - إعطاء أولوية كاملة لمحرك التسجيل.
- إضافة “Adaptive downgrade” أثناء التسجيل:
  - إذا انخفض FPS الفعلي تحت حد معين لعدة ثوانٍ: خفض تلقائي (FPS ثم مؤثرات ثم دقة) دون إفساد التسجيل.

3) الملفات المستهدفة بالتحديث
- src/pages/PreviewPage.tsx
  - ترتيب محركات التسجيل، إصلاح مرجع drawFrame الحي، منطق التدرج التلقائي، وتقليل تحديثات واجهة الحالة أثناء التسجيل.
- src/components/VideoPreview.tsx
  - طبقات الرسم + caching + تخفيف المؤثرات التلقائي + تحسين مسارات الفيديو.
- src/hooks/useVideoRecorder.ts
  - ربط الجودة بالأبعاد الفعلية، ضبط frame pacing، وتحسين إعدادات MediaRecorder حسب الجهاز.
- src/lib/pexelsApi.ts
  - اختيار جودة/أبعاد أكثر ذكاءً بدل أفضل جودة ثابتة.
- supabase/functions/video-proxy/index.ts
  - تحسين proxy ليكون أخف وأكثر توافقًا مع ملفات الفيديو الطويلة.

4) مخاطر متوقعة وكيف سيتم احتواؤها
- اختلاف دعم المتصفحات لخصائص الفيديو/التسجيل:
  - سنحتفظ بمسارات fallback متعددة مع كشف قدرات المتصفح قبل البدء.
- الحفاظ على الشكل “الاحترافي” مع تقليل الحمل:
  - التخفيض سيكون تدريجيًا وذكيًا أثناء الضغط فقط، وليس خفضًا دائمًا للجودة.

5) خطة اختبار شاملة بعد التنفيذ
- اختبارات End-to-End إلزامية:
  1) اختيار فيديو Pexels → معاينة → تسجيل → تنزيل → فحص أن الخلفية متحركة داخل الملف النهائي.
  2) التأكد أن الآيات تتحرك/تتغير مع الصوت بدون توقف.
  3) اختبار على موبايل + ديسكتوب في أوضاع quality/smooth/compatibility/auto.
  4) اختبار مقاطع قصيرة ومتوسطة للتأكد من عدم ظهور تقطيع صوت أو تجميد فيديو.
- قياسات أداء قبل/بعد:
  - زمن الاستجابة، سلاسة المعاينة، استقرار FPS أثناء التسجيل، وحجم الملف النهائي.

6) النتيجة المتوقعة بعد التنفيذ
- اختفاء التهنيج الحاد أثناء التسجيل على معظم الأجهزة.
- توقف مشكلة “الصوت شغال والفيديو واقف”.
- ثبات أفضل لخلفيات Pexels في المعاينة والتسجيل.
- إخراج أخف وأكثر سلاسة مع الحفاظ على الشكل الاحترافي.
